# 🚀 仓库协作指南 (VS Code 版)

## 1. 核心概念预览

在 VS Code 左下角或状态栏，你会看到类似 `main ↓1 ↑2` 的图标：
*   **落后 (Behind) ↓1**：远程仓库有 1 个新提交，你本地分支还没同步（需要 **Pull/拉取**）。
*   **前进 (Ahead) ↑2**：你本地有 2 个新提交，还没推送到远程分支（需要 **Push/推送**）。
*   **同步 (Sync)**：点击循环图标，会自动执行 “拉取再推送”。

## 2. 常用操作流程

### 方案 A：直接在 main 分支修改 (适合小改动/单人)
1.  **拉取最新代码**：点击左下角刷新图标，或在“源代码管理”面板点击 `...` -> `拉取 (Pull)`。确保你的代码是最新的。
2.  **修改代码**：编写你的内容。
3.  **提交 (Commit)**：
    *   在左侧“源代码管理”图标处，看到修改的文件。
    *   点击文件旁的 `+` 号（暂存更改）。
    *   在输入框写下备注（如：`fix: 修复了xx问题`）。
    *   点击 **“提交 (Commit)”** 按钮。
4.  **推送 (Push)**：点击 **“同步更改 (Sync Changes)”** 或 `...` -> `推送 (Push)`。

### 方案 B：创建分支并在本地合并 (适合稍复杂的小改动)
如果你想在一个独立环境下开发，不影响主线：
1.  **创建分支**：点击左下角的 `main` 名字 -> 选择 `+ 创建新分支` -> 输入名字（如 `feature-my-idea`）。
2.  **开发并提交**：按照方案 A 的步骤提交代码。
3.  **合并回 main**：
    *   切换回 `main` 分支（点击左下角分支名选择 `main`）。
    *   点击“源代码管理”面板上方的 `...` -> `分支` -> `合并分支...`。
    *   选择你刚才创建的分支（如 `feature-my-idea`）。
4.  **推送 main**：合并后，点击同步按钮将合并后的 `main` 推送到远程。

### 方案 C：使用 PR (Pull Request) 合并 (最推荐)
如果你希望在合并前让群友看一眼你的代码：
1.  **推送分支**：在你的独立分支（如 `dev`）完成提交后，点击 **“发布分支”**。
2.  **创建 PR**：
    *   方式 1：去 GitHub 网页端，点击醒目的 `Compare & pull request` 按钮。
    *   方式 2：使用 VS Code 的 `GitHub` 插件面板点击 `+` 号创建。
3.  **合并**：在网页端讨论没问题后，点击 `Merge pull request`。
4.  **同步本地**：完成后，回到 VS Code 切换到 `main`，执行 `拉取 (Pull)`。

## 3. 常见问题处理

### Q1: 远程代码更新了，如何把远程的 main 合并到我当前的分支？
当你正在自己的分支开发，但远程 `main` 有了新内容：
1.  在“源代码管理”点击 `...` -> `获取 (Fetch)`（这会下载远程更新，但不会改动你的代码）。
2.  点击 `...` -> `分支` -> `合并分支...`。
3.  选择 **`origin/main`**。
4.  解决冲突（如果有的话），然后提交。

### Q2: 出现冲突 (Conflict) 怎么办？
VS Code 会提示“存在合并冲突”：
1.  点击冲突文件，你会看到 `采用当前更改`、`采用传入更改` 或 `保留双方`。
2.  手动选择或修改代码，直到满意。
3.  点击文件旁的 `+` 号表示已解决。
4.  点击 **提交**。

## 4. 几个好习惯
*   **先拉再写**：每天开始写代码前，先点一下左下角的刷新同步。
*   **小步提交**：每完成一个小功能就提交一次，不要攒一大堆文件再提交。
*   **写清备注**：提交信息简单明了，比如 `add: 增加 readme 说明` 比 `update` 更好。
